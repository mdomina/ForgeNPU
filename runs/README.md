Le run della pipeline scrivono qui gli artifact (`result.json`, RTL seed con moduli come `accumulator_buffer`, testbench, `compiled_program.json` con shape/operator plan, dataflow compilato e flag `preload/transpose`, `verification_vectors.json` con casi modulo, stress top-level e interni, `coverage_report.json`, log dei tool e `execution_report.json` con `dataflow_profile`, queue depth dello scheduler e overlap metrics memoria+compute).
La root puo' contenere anche `dataset/` con archivi JSONL, contatori cumulativi, reward EDA e label accept/reject per il learning loop.
